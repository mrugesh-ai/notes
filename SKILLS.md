# SKILLS.md

## Purpose

This document defines the technical skills and capabilities required to build and maintain the Dev Resource Library project.

It also helps AI agents understand the competencies needed to work effectively with the repository.

---

## Core Development Skills

Developers or AI agents working on this project should be comfortable with:

- HTML
- CSS
- Vanilla JavaScript
- Static website architecture
- DOM manipulation
- Responsive UI design

No backend development is required for the initial version.

---

## Static Website Design

The project is designed as a static site.

Required skills include:

- building multi-page static sites
- dynamic content rendering with JavaScript
- JSON data handling
- client-side navigation

---

## Content Driven Architecture

The website uses a folder-based content system.

Required knowledge:

- JSON data structures
- dynamic UI generation
- folder-based content organization

Example structure:

content/
technology/
category/
resource

Developers must ensure new resources automatically appear in the UI.

---

## Slideshow Implementation

Slides are image collections displayed as presentations.

Required capabilities:

- image loading
- slideshow navigation
- keyboard interaction
- responsive slide layouts

Slides must support:

- next/previous buttons
- keyboard arrow navigation
- slide counter

---

## Performance Optimization

The site must remain fast even as content grows.

Key skills:

- image optimization
- lazy loading
- efficient JavaScript
- minimizing render blocking

---

## UI and UX Design

The project focuses heavily on usability.

Important design skills:

- card-based layouts
- responsive grid design
- simple navigation
- clear resource presentation

The UI should remain minimal and developer-friendly.

---

## Legal Awareness

Since the site organizes third-party resources, contributors must understand:

- copyright considerations
- content attribution
- removal request handling

The project must include:

- disclaimer page
- privacy policy
- content removal process

---

## Maintenance Workflow

Contributors should understand the content workflow:

1. add files to `/content`
2. update `content.json`
3. commit changes
4. website automatically updates

The goal is minimal maintenance.

---

## Future Skills

Future improvements may require knowledge of:

- client-side search
- indexing
- tagging systems
- static site optimization
- migration to frameworks if needed

However, the current system should remain framework-free.
